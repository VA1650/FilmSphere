# FilmSphere — Fullstack Movie & Playlist Management Service

Веб-сервис для каталогизации фильмов и управления кастомными плейлистами с поддержкой многопользовательского контекста. Приложение реализовано как модульное клиент-серверное решение (SPA) со своим выделенным бэкендом и локальной реляционной базой данных.

## 🚀 Технологический стек

* **Frontend:** React 19 (Hooks, State-based routing, Functional Components), HTML5/CSS3 (компоненты с адаптивной версткой).
* **Backend:** Node.js, Express (REST API, CORS middleware).
* **Database:** SQLite3 (реляционная БД, постоянное хранение данных, поддержка связей между таблицами).

## 🏗 Архитектура и ключевые фичи

* **Архитектурный паттерн Client-Server:** Проект полностью автономен. Клиентская часть изолирована от бэкенда и взаимодействует с ним исключительно через асинхронные REST-запросы (`fetch`).
* **Концепция Profile Switching (Многопользовательский режим):** Реализована сквозная логика работы для нескольких учетных записей без жесткой перегрузки интерфейса формами авторизации. Выбор пользователя динамически переключает контекст базы данных, изолируя плейлисты и персональные подборки одного юзера от другого.
* **State-based Навигация:** Роутинг внутри Single Page Application (SPA) реализован через централизованную машину состояний (`currentView`). Это гарантирует мгновенный рендеринг экранов без лишних перезагрузок страниц и минимизирует пинг интерфейса.
* **Полный цикл CRUD операций:** Система поддерживает реактивное изменение данных на лету:
  * Создание / удаление пользовательских плейлистов.
  * Динамическое добавление и деструктуризация (удаление) фильмов внутри конкретных выборок.

## 📱 Структура экранов приложения

1. **Главный экран (Каталог):** Агрегатор всех доступных в системе кинокартин с возможностью перехода на детальные карточки.
2. **Панель плейлистов:** Пространство, отображающее списки воспроизведения, привязанные к текущему активному профилю.
3. **Экран подборки (List Detail):** Детальный просмотр фильмов внутри выбранного плейлиста с интерфейсом добавления/удаления позиций.
4. **Карточка фильма (Film Card):** Экран с расширенным мета-описанием кинокартины и триггером экспорта в пользовательские списки.

## 🔧 Запуск локального окружения

### 1. Клонирование и установка зависимостей
```bash
# Установка пакетов для всего проекта (включая бэкенд и фронтенд компоненты)
npm install
```
### 2. Запуск сервера базы данных и бэкенда

Серверная часть разворачивается на порту 5000 и обрабатывает API-запросы:
```bash

node server.js 
```

3. Запуск клиентской части (React Development Server)
```bash

npm start
```
После этого клиентская часть автоматически откроется в браузере по адресу http://localhost:3000.
